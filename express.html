Express.js

Framework minimalis untuk Node.js yang memudahkan pembuatan server dan API. Pelajari routing, middleware, dan struktur dasar proyek Express.

Apa itu Express.js?

Express.js adalah framework web untuk Node.js yang menyediakan fitur-fitur sederhana namun powerful untuk membangun aplikasi web dan API. Express menggunakan pola middleware yang memungkinkan ekstensi fungsi dengan mudah.

Keunggulan Express.js dibanding Node.js biasa:

  • Routing yang sederhana: Sistem routing yang mudah dipahami
  • Middleware: Fleksibilitas untuk menambahkan fungsi tambahan
  • Ringan dan cepat: Performa tinggi dengan overhead minimal
  • Ekosistem besar: Banyak middleware dan plugin tersedia
  • Kompatibilitas: Bisa digunakan dengan berbagai template engine
  • Komunitas aktif: Dokumentasi lengkap dan dukungan komunitas

Kapan menggunakan Express.js?

  • Membangun API RESTful
  • Aplikasi web sederhana hingga menengah
  • Microservices
  • Prototyping cepat
  • Sebagai dasar untuk framework lain (Nest.js, etc)

Instalasi Express.js

Untuk mulai menggunakan Express.js, Anda perlu menginstal Node.js terlebih dahulu, kemudian membuat proyek Node.js dan menginstal Express.

Langkah-langkah instalasi:

  1. Download dan install Node.js dari nodejs.org
  2. Buka terminal/command prompt
  3. Buat folder proyek baru dan masuk ke folder tersebut:
    mkdir proyek-express
    cd proyek-express
  4. Inisialisasi proyek Node.js:
    npm init -y
  5. Instal Express:
    npm install express
  6. Buat file utama (biasanya index.js atau app.js):
    touch index.js

Persyaratan Sistem:

  • Node.js ≥ 14.x
  • NPM atau Yarn
  • Text editor atau IDE
  • Terminal/command prompt

Server Express Dasar

Berikut contoh membuat server dasar dengan Express.js yang merespon di port 3000.

// index.js
const express = require('express');
const app = express();
const port = 3000;

// Route dasar
app.get('/', (req, res) => {
  res.send('Halaman Utama');
});

// Jalankan server
app.listen(port, () => {
  console.log(`Server berjalan di http://localhost:${port}`);
});

Menjalankan Server:

# Jalankan server
node index.js

# Dengan nodemon (auto restart saat perubahan)
npx nodemon index.js

Catatan: Untuk menggunakan nodemon, instal terlebih dahulu dengan:
npm install -g nodemon atau npm install --save-dev nodemon

Routing di Express

Express menyediakan metode routing untuk berbagai jenis HTTP request. Berikut contoh routing dasar untuk GET, POST, PUT, dan DELETE.

// GET request
app.get('/produk', (req, res) => {
  res.send('Daftar produk');
});

// POST request
app.post('/produk', (req, res) => {
  res.send('Produk baru ditambahkan');
});

// PUT request
app.put('/produk/:id', (req, res) => {
  res.send(`Produk ${req.params.id} diperbarui`);
});

// DELETE request
app.delete('/produk/:id', (req, res) => {
  res.send(`Produk ${req.params.id} dihapus`);
});

Parameter Request:

  • req.params - Parameter dari URL (contoh: /produk/:id)
  • req.query - Query string (?nama=value&harga=100)
  • req.body - Data yang dikirim dalam body request (perlu middleware)
  • req.headers - HTTP headers dari request

Middleware

Middleware adalah fungsi yang memiliki akses ke request object (req), response object (res), dan fungsi next() dalam siklus request-response. Middleware dapat:

  • Menjalankan kode apa pun
  • Mengubah request dan response objects
  • Mengakhiri siklus request-response
  • Memanggil middleware berikutnya dalam stack
// Middleware bawaan untuk parsing JSON
app.use(express.json());

// Middleware logger sederhana
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next();
});

// Middleware untuk route tertentu
app.use('/admin', (req, res, next) => {
  console.log('Akses ke area admin');
  next();
});

Middleware Populer:

  • express.json() - Parsing body JSON
  • express.urlencoded() - Parsing form data
  • cors - Mengatasi CORS
  • helmet - Keamanan HTTP headers
  • morgan - Logger HTTP request
  • cookie-parser - Parsing cookies

Struktur Folder Proyek

Berikut struktur folder yang umum digunakan dalam proyek Express.js:

nama-proyek/
├── node_modules/ # Dependencies
├── controllers/ # Logic aplikasi
├── routes/ # File routing
├── middleware/ # Custom middleware
├── models/ # Model database
├── public/ # File statis
│ ├── css/
│ ├── js/
│ └── images/
├── views/ # Template views
├── .env # Environment variables
├── package.json # Konfigurasi proyek
└── app.js # File utama

Best Practices:

  • Pisahkan routing, controller, dan model
  • Gunakan environment variables untuk konfigurasi sensitif
  • Gunakan middleware untuk fungsi umum
  • Gunakan template engine untuk views (EJS, Pug, Handlebars)
  • Gunakan dotenv untuk mengelola environment variables

Contoh API Sederhana

Berikut contoh implementasi API sederhana dengan Express.js untuk operasi CRUD.

// app.js
const express = require('express');
const app = express();
app.use(express.json());

// Data sementara
let produk = [
  { id: 1, nama: 'Laptop', harga: 8000000 },
  { id: 2, nama: 'Smartphone', harga: 4000000 }
];

// GET semua produk
app.get('/api/produk', (req, res) => {
  res.json(produk);
});

// GET produk by id
app.get('/api/produk/:id', (req, res) => {
  const produkItem = produk.find(p => p.id === parseInt(req.params.id));
  if (!produkItem) return res.status(404).send('Produk tidak ditemukan');
  res.json(produkItem);
});

// POST tambah produk baru
app.post('/api/produk', (req, res) => {
  const produkBaru = {
    id: produk.length + 1,
    nama: req.body.nama,
    harga: req.body.harga
  };
  produk.push(produkBaru);
  res.status(201).json(produkBaru);
});

// Jalankan server
const port = process.env.PORT || 3000;
app.listen(port, () => console.log(`Server berjalan di port ${port}`));
