Node.js

Platform runtime JavaScript untuk membuat aplikasi backend yang cepat dan efisien. Pelajari cara kerja server, routing, dan penggunaan Express.js.

Apa itu Node.js?

Node.js adalah runtime environment JavaScript yang dibangun di atas mesin JavaScript V8 Chrome. Node.js memungkinkan JavaScript dijalankan di sisi server, bukan hanya di browser.

Keunggulan Node.js untuk backend web:

  • Non-blocking I/O: Mampu menangani banyak request secara bersamaan
  • JavaScript di server: Gunakan bahasa yang sama di frontend dan backend
  • Ekosistem npm: Perpustakaan terbesar di dunia dengan ribuan package
  • Ringan dan cepat: Dibangun di atas V8 yang sangat cepat
  • Komunitas besar: Dokumentasi lengkap dan dukungan luas

Kapan menggunakan Node.js?

  • Aplikasi real-time (chat, game online)
  • API dan microservices
  • Aplikasi dengan banyak I/O operations
  • Single Page Applications (SPA)
  • Streaming aplikasi

Instalasi Node.js dan npm

Untuk mulai menggunakan Node.js, Anda perlu menginstalnya terlebih dahulu di komputer Anda.

Langkah-langkah instalasi:

  1. Download Node.js dari nodejs.org
  2. Pilih versi LTS (Long Term Support) yang direkomendasikan
  3. Jalankan installer dan ikuti petunjuk instalasi
  4. Verifikasi instalasi dengan perintah di terminal:
    node -v
    npm -v

npm (Node Package Manager) akan terinstal otomatis bersama Node.js. npm adalah alat untuk mengelola package JavaScript.

Tips untuk Pemula:

  • Gunakan text editor seperti VS Code, Sublime Text, atau WebStorm
  • Biasakan menggunakan npx untuk menjalankan package lokal
  • Pelajari dasar terminal/command line
  • Gunakan nodemon untuk auto-restart server saat development

Menjalankan File JavaScript

Setelah Node.js terinstal, Anda bisa menjalankan file JavaScript langsung dari terminal.

# Buat file dengan ekstensi .js, contoh: app.js
console.log("Hello, Node.js!");

# Jalankan di terminal:
node app.js

Contoh File JavaScript

// app.js
const nama = "Budi";
const umur = 25;

function sapa(nama) {
  return `Halo, ${nama}!`;
}

console.log(sapa(nama));
console.log(`Umur Anda: ${umur} tahun`);

Membuat Server dengan HTTP

Node.js memiliki modul HTTP bawaan untuk membuat server web tanpa framework tambahan.

// server.js
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello from Node.js server!');
});

server.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});

Menjalankan Server:

# Jalankan di terminal:
node server.js

# Buka di browser:
http://localhost:3000

Tekan Ctrl+C di terminal untuk menghentikan server.

Pengenalan Express.js

Express.js adalah framework web untuk Node.js yang menyederhanakan pembuatan server dan routing. Express adalah framework backend yang paling populer di ekosistem Node.js.

Instalasi Express:

# Buat folder proyek baru
mkdir my-express-app
cd my-express-app

# Inisialisasi proyek Node.js
npm init -y

# Install Express
npm install express

Struktur dasar proyek Express:

proyek_express/
├── node_modules/ # Dependencies
├── package.json # Konfigurasi proyek
├── app.js # File utama aplikasi
└── .gitignore # File yang diabaikan Git

Contoh aplikasi Express minimal:

// app.js
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello Express!');
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});

Routing dengan Express

Routing adalah cara Express memetakan URL ke fungsi yang akan menangani request tersebut.

const express = require('express');
const app = express();

// Route dasar
app.get('/', (req, res) => {
  res.send('Halaman Utama');
});

// Route dengan parameter
app.get('/user/:id', (req, res) => {
  res.send(`User ID: ${req.params.id}`);
});

// Route dengan method POST
app.post('/login', (req, res) => {
  // Proses data login
  res.send('Login berhasil');
});

// Middleware untuk parsing JSON
app.use(express.json());

app.listen(3000);

Contoh Form dengan Express

// app.js
const express = require('express');
const app = express();

// Middleware untuk parsing form data
app.use(express.urlencoded({ extended: true }));

app.get('/form', (req, res) => {
  res.send(`
    <form method="POST" action="/submit">
      <input type="text" name="nama" placeholder="Nama Anda">
      <button type="submit">Kirim</button>
    </form>
  `);
});

app.post('/submit', (req, res) => {
  res.send(`Halo, ${req.body.nama}!`);
});

app.listen(3000);

Membuat API Sederhana

Node.js dengan Express sangat cocok untuk membuat API backend. Berikut contoh membuat API sederhana.

// app.js
const express = require('express');
const app = express();

// Middleware untuk parsing JSON
app.use(express.json());

// Data dummy
let products = [
  { id: 1, name: "Laptop", price: 1000 },
  { id: 2, name: "Smartphone", price: 500 }
];

// GET semua produk
app.get('/api/products', (req, res) => {
  res.json(products);
});

// GET produk by ID
app.get('/api/products/:id', (req, res) => {
  const product = products.find(p => p.id === parseInt(req.params.id));
  if (!product) return res.status(404).send('Produk tidak ditemukan');
  res.json(product);
});

// POST tambah produk baru
app.post('/api/products', (req, res) => {
  const product = {
    id: products.length + 1,
    name: req.body.name,
    price: req.body.price
  };
  products.push(product);
  res.status(201).json(product);
});

app.listen(3000, () => console.log('API running on port 3000'));

Tips untuk API Node.js:

  • Gunakan Express Router untuk mengorganisir route
  • Validasi input dari client
  • Gunakan status code HTTP yang tepat
  • Pertimbangkan untuk menggunakan TypeScript
  • Dokumentasikan API dengan Swagger atau Postman

Struktur Proyek Node.js

Berikut struktur proyek Node.js yang direkomendasikan untuk pemula:

my-node-app/
├── node_modules/ # Dependencies
├── controllers/ # Logic bisnis
│ └── productController.js
├── routes/ # File routing
│ └── productRoutes.js
├── models/ # Model data
│ └── Product.js
├── config/ # Konfigurasi
│ └── db.js
├── public/ # File statis
│ ├── css/
│ └── js/
├── app.js # File utama
├── package.json # Dependencies
└── .env # Environment variables

Best Practices:

  • Gunakan dotenv untuk environment variables
  • Pisahkan concern dengan MVC pattern
  • Gunakan middleware untuk logging dan error handling
  • Gunakan ESLint dan Prettier untuk konsistensi kode
  • Gunakan version control (Git) dari awal
